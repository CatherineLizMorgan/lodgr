# üìù 4. Commit Message Guidelines

Types:
- feat  ‚Äì new feature
- fix   ‚Äì bug fix
- refactor ‚Äì structural improvement, no behavior change
- docs ‚Äì documentation changes
- style ‚Äì formatting only
- chore ‚Äì maintenance tasks
- test ‚Äì adds or updates tests

# üé® 5. Code Style Standards

### Frontend (React/Next.js)
- Use functional components + hooks  
- Prefer TypeScript  
- Keep components modular  
- Follow project linting (ESLint + Prettier)

### Backend (Node.js/NestJS/Express)
- Use async/await
- Validate all inputs
- Provide descriptive error handling
- Use environment variables (never commit `.env`)

### Database
- Write safe, reversible migrations  
- Document schema changes in `/docs/schema.md`
